Q1. What dose JavaScript Event Loop do?

JavaScript is a single threaded language. That's why it can do one task at a time. It has a call stack and an event queue. When JavaScript call stack or single thread execution unit executing a task and at the same time it need's to execute another task it doesn't stop what it was doing. It adds the second task to the event queue which is a first in first out ordering of event's and wait until the first task is done. After finishing the first task event queue push the second task to the call stack. This is how it maintains what it has to do next and this is what an event loop does in JavaScript.

Q2. What is the difference between LocalStorage and SessionStorage


LocalStorage


  1. LocalStorage stores key-value pares for a particular website and it stores the value only in string.
  2. LocalStorage can use up to 10MB of storage and this data isn't sent back to the server every time the client visits the website.
  3. This data stays permanently.

SessionStorage


  1. SessionStorage also stores key-value pares for a particular website and its store value only in the sting.
  2. SessionStorage can use up to 5MB of storage.
  3. This data is temporary. SessionStorage stores data as long as the particular webpage or tab is open. After closing the webpage or tab this data is deleted.
